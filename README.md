# 协纪 · 黄历择吉工具

> 「协纪」之名取自清官修《协纪辨方书》——传统择吉典籍中最权威的一部。

基于 lunar-python + Streamlit 的中国传统择吉系统。把建除十二神、黄黑道、二十八宿、神煞宜忌、八字喜用、生肖冲合、方位太岁、紫微斗数等十多个传统维度，整合成可交互的 Web 工具。

---

## 快速开始

```bash
pip install -r requirements.txt
streamlit run app.py
```

打开 `http://localhost:8501`。

如要启用 LLM 深度解读：

```bash
export ANTHROPIC_API_KEY=sk-ant-...
streamlit run app.py
```

或在侧边栏临时输入 API key。无 key 时自动降级为规则式解读。

### 中文字体（PDF / PNG 导出必备）

PDF 老黄历与分享海报 PNG 都需要本机存在 **中文字体**，否则会输出"豆腐方块"。仓库内置候选路径已覆盖 macOS / Windows 常见字体，**Linux 需手动装一次**：

```bash
# Debian / Ubuntu（含 GitHub Codespaces / VS Code devcontainer）

# CentOS / RHEL / Fedora

```

devcontainer 用户**无需手动操作**：仓库根目录的 `packages.txt` 会在容器首次启动时自动 `apt install`。

## 各模块实现原理

### 1. `almanac.py` — 日级黄历核心

封装 lunar-python 的 `Lunar` / `Solar` 类，把单日所有黄历要素汇总到一个 `DayAlmanac` dataclass。

**输出字段**：
- 公历日期、农历表达、年/月/日干支、生肖年
- 日干、日支、各自五行（用内置的 `GAN_WUXING` / `ZHI_WUXING` 查找表）
- 纳音五行、建除十二神（`getZhiXing`）
- 黄黑道当值神（`getDayTianShen`）+ 黄道/黑道分类
- 二十八宿 + 吉凶判定（`getXiu` / `getXiuLuck`）
- 通书宜忌（`getDayYi` / `getDayJi`）
- 吉神/凶煞列表（`getDayJiShen` / `getDayXiongSha`）
- 冲煞、煞方、彭祖百忌、喜神/财神方位

**关键查找表**：
- `HUANGDAO_STARS` / `HEIDAO_STARS` — 黄黑道十二神分类
- `JIANCHU_LUCKY` / `JIANCHU_BAD` — 建除十二神吉凶分类
- `MAJOR_JISHEN` / `MAJOR_XIONGSHA` — 重要吉神凶煞集合（评分加权用）
- `ZHI_CHONG` / `ZHI_SANHE` / `ZHI_LIUHE` / `ZHI_HAI` — 地支六冲/三合/六合/相害关系

`get_birth_chart(dt)` 是简化版接口，只取日干/日支/年生肖等关键字段，供 scoring 使用。

---

### 2. `hours.py` — 12 时辰择吉与真太阳时校正

**时辰盘**：调 `Lunar.getTimes()` 返回 13 个 `LunarTime` 对象（晚子时单算），逐个提取：
- 时辰干支（基于"五鼠遁日起时"自动推算）
- 当值天神 + 黄黑道分类
- 时辰冲煞生肖
- 喜神/财神/福神/阳贵/阴贵 5 个方位
- 时辰宜忌（lunar-python 自带）

**时辰评分**（0-100）：基础 50 → 黄道 +15 / 黑道 −15 → 天神吉 +5 / 凶 −5 → 旬空 −10。

**真太阳时校正**：
```
真太阳时 = 北京时间 + (本地经度 − 120°) × 4 分钟 + 时差方程
```
- 经度修正：每偏 1° 差 4 分钟。乌鲁木齐 87.62° 距北京 32.4° → 约 −130 分钟。
- 时差方程（Equation of Time）：太阳赤纬与轨道偏心率引起的偏差，用 Spencer 近似公式 `9.87·sin(2B) − 7.53·cos(B) − 1.5·sin(B)`，其中 `B = 2π·(day_of_year − 81)/364`。误差年内 ±16 分钟。

预置 34 个城市经度，用户选城市即自动校正八字时柱。

### 3. `bazi.py` — 完整四柱八字排盘

调 lunar-python 的 `EightChar` 对象拿到原始数据：
- 四柱干支（自动用立春切年、节气切月）
- 天干十神、地支十神、藏干、纳音、旬、旬空
- 胎元、命宫、身宫
- 大运（按性别 1/0 分阴阳顺逆排）

**五行强弱判定**（简化算法）：
```
score = 0
# 1. 得令：月支当令五行 vs 日主
if 月令五行 == 日主：score += 3
elif 生我：score += 2
elif 我生：score -= 1
elif 克我：score -= 1
elif 我克：score += 1

# 2. 通根：其余三支同五行
for 支 in (年/月/时支)：
    if 同我：score += 2
    if 藏干同我：score += 0.5（每个）

# 3. 同党：其余天干同五行或生我
for 干 in (年/月/时干)：
    if 同我：score += 1
    elif 生我：score += 0.5

# 等级
≥6  → 偏强
≥4  → 中和偏强
≥2  → 中和
≥0  → 中和偏弱
<0  → 偏弱
```

**喜用神推断**（简化 5 行法则）：
- 偏弱 → 用印（生我）+ 比劫（同我）；忌食伤/财/官杀
- 偏强 → 用食伤/财/官杀；忌印/比劫
- 中和 → 取八字五行计数最少的两个为调候用神

**合婚评分** `hehun_score(bride, groom)`：基础 50，加减项：
- 生肖六冲 −25 / 六合 +15 / 三合 +12 / 相害 −10
- 日支同 +5 / 冲 −15 / 六合 +8
- 喜用神共通 +6
- 一方喜用神是对方强项（≥2）+ 自身缺（<1）→ +4 "互补"

最终分级：上等/中上/中等/下中/不宜婚。

### 4. `directions.py` — 方位与九宫

**太岁/岁破/三煞** 按年支推算：
- 太岁 = 当年年支所在方位（子=正北、丑=北偏东...）
- 岁破 = 太岁对宫（子↔午、丑↔未...）
- 三煞 = 太岁三合局的对宫三支
  - 申子辰水局 → 三煞在南方（巳午未）
  - 寅午戌火局 → 三煞在北方（亥子丑）
  - 亥卯未木局 → 三煞在西方（申酉戌）
  - 巳酉丑金局 → 三煞在东方（寅卯辰）

**忌动山向** = 太岁 + 岁破 + 三煞三支去重；**宜动山向** = 与太岁三合的另外两支。

**二十四山向**：8 干（戊己居中不取）+ 12 支 + 4 维（乾坤艮巽），按罗盘顺序：壬子癸丑艮寅甲卯乙辰巽巳丙午丁未坤申庚酉辛戌乾亥。

**本命卦**（八宅派）：
- 上元（1864-1923）/ 中元（1924-1983）/ 下元（1984- ）划分
- 男女算法不同（民间通行公式）：
  ```
  if year < 2000:
      男：n = (100 − 年后两位) % 9
      女：n = (年后两位 + 4) % 9
  else:
      男：n = (99 − 年后两位) % 9
      女：n = (年后两位 + 5) % 9
  ```
- 数字 → 卦：1坎 2坤 3震 4巽 6乾 7兑 8艮 9离 （5 男为坤、女为艮）
- 东四命（坎离震巽）/ 西四命（乾坤艮兑）
- 每卦对应 4 吉方（生气/天医/延年/伏位）+ 4 凶方（祸害/六煞/五鬼/绝命），用查找表存储

**九宫飞星**：取 `Lunar.getYearNineStar()` 拿到中宫数字，按洛书顺飞顺序"中→西北→正西→东北→正南→正北→西南→正东→东南"填入 9 个方位。再用 `STAR_LUCK` 表标注每星的吉凶（如 5 黄廉贞=大凶、8 白左辅=大吉）。

### 5. `calendars.py` — 节令与特殊日历

**二十四节气精确时刻**：`Lunar.getJieQiTable()` 返回 30+ 个节气（含跨年），每个 `Solar` 含年月日时分秒。按 `JIEQI_ORDER` 顺序输出。

**三伏**（夏季三个庚日）：
- 初伏 = 夏至后第 3 个庚日（10 天）
- 中伏 = 夏至后第 4 个庚日（10 或 20 天，取决于末伏时点）
- 末伏 = 立秋后第 1 个庚日（10 天）

实现：遍历日期查 `Lunar.getDayGan() == '庚'`。

**数九**：冬至起 9 个九天，共 81 天。从冬至 Solar 开始递增 9 天即可。

**月相**：直接调 `Lunar.getYueXiang()`（既朔/望/晓 等）。

**法定节假日**：硬编码 2026-2027 国务院公告表（春节/清明/端午/中秋/国庆等），便于查询时直接显示"5-1 劳动节"。

### 6. `rules.py` — 传统凶日规则表

逐条实现"硬规则"凶日判定，每条返回 bool。`evaluate_taboos()` 一次性返回所有命中。

| 三娘煞 | 农历初三、初七、十三、十八、廿二、廿七 | 集合查找 |
| 十恶大败 | 甲辰/乙巳/丙申/丁亥/戊戌/己丑/庚辰/辛巳/壬申/癸亥 | 干支字符串集合 |
| 杨公十三忌 | 农历正月十三、二月十一……十二月十九（七月还有廿九） | `{月: 日}` 查表 |
| 月忌日 | 农历初五、十四、廿三 | 集合查找 |
| 月厌 | 月支 → 厌日地支查表 | 寅戌、卯酉、辰申…… |
| 红沙日 | 孟月（寅巳申亥）逢酉 / 仲月（卯午酉子）逢巳 / 季月（辰未戌丑）逢丑 | 集合判定 |
| 重丧日 | 月支 → 日干查表（如寅月甲日、卯月乙日） | 字典查找 |
| 归忌 | 季节 → 日支（春丑/夏寅/秋子/冬卯） | 月支映射 |

### 7. `scoring.py` — 综合评分与推荐

`score_day(almanac, event, persons, school)` 是核心函数，基础分 50，8 大维度加减：

```
1. 黄黑道当值        ± huangdao
2. 建除十二神        + jianchu_lucky / + 0 / − jianchu_bad
3. 二十八宿吉凶      ± xiu
4. 吉神（重要+少量） + major_jishen / + minor_jishen
5. 凶煞（重要+少量） − major_xiongsha / − minor_xiongsha
6. 事项宜/忌匹配     + yi_match (20) / − ji_match (30, fatal)
7. 多当事人冲煞      − chong_birth (fatal) / + sanhe / + liuhe / − hai
8. 日柱五行 vs 日主  生我+3 / 比和+2 / 我克+1 / 我生−1 / 克我−3
9. 传统凶日命中      事项相关凶日 −15（部分触发 fatal）
```

**Fatal 机制**：命中"诸事不宜"级凶日（月破、岁破、四离四绝、灭门、受死、冲日主等）会标 `fatal=True`，从推荐结果中剔除。

**流派权重**（`SCHOOL_WEIGHTS`）：
- **综合**：默认平衡配置
- **协纪辨方**：神煞权重最高（清官修《协纪辨方书》风格）
- **董公选日**：宜忌权重最高，神煞次之（民间通行）
- **宽松模式**：所有惩罚减半，不一票否决（适合非传统人士）

**推荐排序** `recommend_days()`：
1. 剔除所有 fatal
2. 排序键：`(明确"宜"本事 > 未明确, raw_score 降序, 日期升序)`
3. `raw_score` 不封顶（防止多个 100 分日子无法区分）；展示分仍封顶 100

**多当事人**：`persons` 参数支持 1-4 个人，每人独立检查冲日支与喜用神匹配。任何一人冲日主即整日 fatal。

### 8. `wedding.py` — 嫁娶周堂图

8 个位置顺时针排列：`夫 → 姑 → 堂 → 翁 → 第 → 灶 → 妇 → 厨`，其中"夫妇翁姑"为凶位（对应人需短暂回避礼堂），"堂第灶厨"为吉位。

**算法**：
- 大月（30 天）：从【夫】（index 0）起顺数到农历日数
- 小月（29 天）：从【妇】（index 6）起逆数到农历日数

```python
if is_big_month:
    idx = (lunar_day - 1) % 8        # 顺数
else:
    idx = (6 - (lunar_day - 1)) % 8  # 从妇逆数
```

**月大小判定**：尝试 `Lunar.fromYmd(year, month, 30)`，能构造则是大月，抛错则是小月。

`avoid_persons_for_zhoutang(pos)` 返回该位需避之人（夫→新郎、妇→新娘、翁→新郎父、姑→新郎母）。

### 9. `dayun.py` — 大运/流年深度评估

**单步大运评分**（基础 50）：
- 大运天干五行 vs 日主：是喜用神 +12 / 忌神 −10 / 比肩 +6
- 大运地支五行 vs 日主：喜用 +14 / 忌神 −12（地支力量大于天干）
- 大运地支 vs 日支：冲 −15 / 合 +8 / 三合 +6
- 大运地支 vs 月令：同 +4 / 冲 −8

最终等级：佳/顺/平/蹇/厄运。

**Narrative**：按地支/天干是否为喜用神生成 2-3 句白话评语。例如：
> "此运地支为喜用神，根基稳固，凡事易得贵人助力。天干透出忌神，外缘多扰，宜低调行事。综合而言，此运中平，常事可行，大事宜审慎。"

**流年评估** `evaluate_liunian(bz, dayun, year)`：
- 流年干支五行匹配（+6/−5/+8/−7）
- 流年支冲日支 −10
- 流年支冲大运支 −12（重大转折）
- 流年与大运三合 +5
- 值太岁本命年 −8、冲太岁 −10

### 10. `ziwei.py` — 紫微斗数

**命宫定位**：
1. 从【寅】（index 2）起，顺时针数到农历月数（正月寅、二月卯…）
2. 从该位置逆时针数生时支
3. 落点即命宫地支

```python
month_pos = (2 + (lunar_month - 1)) % 12
ming_pos = (month_pos - time_zhi_idx) % 12
```

**命宫天干**（五虎遁年起月法）：
- 年干甲己 → 寅月起丙、年干乙庚 → 起戊、年干丙辛 → 起庚、年干丁壬 → 起壬、年干戊癸 → 起甲
- 从寅起到命宫支偏移 N 位 → 命宫天干 = 起干 + N

**五行局**：命宫干支查 60 甲子纳音表，对应：水二局/木三局/金四局/土五局/火六局。

**紫微星定位**：标准表查 `ZIWEI_TABLE[局数][农历日 - 1]`，返回所落地支。表为传统通行的 5×30 矩阵，例如：
```
水二局 day1-30: 丑 寅 寅 卯 卯 辰 辰 巳 巳 午 ...
火六局 day1-30: 酉 午 亥 辰 丑 寅 戌 未 子 巳 ...
```

**天府星**：紫微的镜像，`(4 − 紫微 index) % 12`。验证：紫微在子(0)时天府在辰(4)，紫微在寅(2)时天府在寅(2)，紫微在亥(11)时天府在巳(5) → 始终满足两者之和 ≡ 4 (mod 12)。

**14 主星定位**：
- 紫微系（逆数）：紫微 → 天机(−1) → 太阳(−3) → 武曲(−4) → 天同(−5) → 廉贞(−8)
- 天府系（顺数）：天府 → 太阴(+1) → 贪狼(+2) → 巨门(+3) → 天相(+4) → 天梁(+5) → 七杀(+6) → 破军(+10)

**12 宫排列**：从命宫起逆时针排：命宫 → 兄弟 → 夫妻 → 子女 → 财帛 → 疾厄 → 迁移 → 奴仆 → 官禄 → 田宅 → 福德 → 父母。每宫所含主星 = 该地支位置上的所有星。

**命宫批注**：14 主星各有特性短语（如紫微 = "帝王之星，主刚强果断"、太阴 = "母星，主温柔细腻"），命宫坐星直接展开为人话。

### 13. `output/llm.py` — 多提供商 LLM 自然语言解读

支持两套 SDK，按 provider 分发：

| Provider | 用 | 适用端点 |
|---|---|---|
| `anthropic` | anthropic SDK | Claude 原生 + 任何 Anthropic 协议兼容代理 |
| `openai` | openai SDK | OpenAI 原生 + DeepSeek / Moonshot / 智谱 / 通义 / Ollama / vLLM 等兼容端点 |

**预设端点**（侧边栏可一键选择）：

OpenAI 兼容：
- OpenAI 官方（默认 `api.openai.com/v1`）
- DeepSeek `https://api.deepseek.com`
- Moonshot (Kimi) `https://api.moonshot.cn/v1`
- 智谱 GLM `https://open.bigmodel.cn/api/paas/v4`
- 通义千问 `https://dashscope.aliyuncs.com/compatible-mode/v1`
- Ollama 本地 `http://localhost:11434/v1`
- vLLM 本地 `http://localhost:8000/v1`
- 自定义 `https://任意 OpenAI 兼容端点`

每个端点配套推荐模型列表（如 DeepSeek → `deepseek-chat` / `deepseek-reasoner`），也支持手填自定义模型名。

**核心接口**：
```python
@dataclass
class LLMConfig:
    provider: str       # "anthropic" | "openai"
    model: str          # 模型名
    api_key: str        # 必填
    base_url: str = ""  # 留空 = 用 SDK 默认

interpret_via_llm(result, event=..., persons=..., config=LLMConfig(...))
```

**Prompt Caching**：
- Anthropic：系统提示打 `cache_control: {type: "ephemeral"}`，5 分钟 TTL
- OpenAI 兼容：无显式 cache 字段；OpenAI 原生有隐式 prefix cache，其他兼容端点视实现而异

**降级策略**（任何一步失败都返回规则式 `natural_language_summary`，并附错误简要）：
1. `config is None` 且无环境变量 key → 直接降级
2. `config.is_usable() == False`（空 key 或空 model）→ 直接降级，不发请求
3. SDK ImportError → 降级 + 提示装哪个包
4. 任何 API 异常（认证/速率/网络/超时/未知 host）→ 降级 + 附 `异常类型: 简要消息`

侧边栏配置（5 个字段）：
- API 类型（Anthropic / OpenAI 兼容）
- 端点 / 服务商（预设下拉）
- Base URL（选自定义时显示）
- API Key
- 模型（按端点动态切换预设列表 + 自定义输入）

## 限制与说明

- **范围**：lunar-python 支持 1900-2100 年，超出此范围可能不准。
- **闰月**：lunar-python 用负数月（如 `-3` 表示闰三月）表示，闰月当月评分按相同 `month_zhi` 处理，无特殊神煞。
- **真太阳时**：仅做经度 + 时差方程修正，未考虑历史时区调整（夏令时/民国时间等）。1991 年前的中国出生者用现代北京时间反推会有偏差。
- **八字强弱判定**：本工具用简化算法（得令 + 通根 + 同党加分），传统命理还需考虑"调候""通关""从格"等高级判法，重大事项请咨询专业命理师。
- **紫微斗数**：仅实现命宫定位 + 14 主星 + 命宫批注，未覆盖副星（左辅右弼/文昌文曲/六煞）、化禄化忌、限运、宫干四化等深度内容。
- **不将日**：完整表很长，本工具只录入每月代表性几个，覆盖率约 70%。
- **本工具是文化参考工具，不是命定预测**。重大事项（婚丧嫁娶/动土上梁/重大投资）建议结合专业课择 + 当事人完整八字综合判断。
